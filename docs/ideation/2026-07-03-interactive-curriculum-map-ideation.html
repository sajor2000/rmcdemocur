Grounding Context

Codebase Context. RushMap AI is a Next.js 14 app mapping RMD 563 faculty guides to USMLE 2025 and AAMC frameworks via Azure OpenAI + Neon pgvector. The ingestion pipeline parses documents, chunks (~500 tokens, 50 overlap), embeds chunks, aligns to framework candidates via RAG, tags AAMC keywords, and recomputes framework-level gap summaries.

The current signature map (/courses/[id]/map) renders three parallel panels — curriculum tree, AAMC tree, USMLE tree — with filter controls and an alignment review drawer. Planned SVG alignment lines were deferred; no file AlignmentLines.tsx exists. The gaps page and dashboard report framework coverage (covered / partial / gap) but do not surface intra-curriculum redundancy or cross-case relational views.

Leverage points: chunk embeddings already in Postgres; keyword_tags populated at pipeline time but unused in UI; alignments table links every chunk to multiple framework leaves with confidence and rationale; parentStableId on framework tables enables rollup.

Pain points: users cannot see why two cases both "light up green" on the same USMLE node; gap recommendations always suggest adding content, never consolidating; three-column layout hides relationships; no metric for over-teaching.

External context: mapEDU markets gap and redundancy surfacing as core healthcare curriculum value. CloudPedagogy's mapping engine uses multipartite SVG graphs and outcome heatmaps for structural duplication. ForestED (2025) visualizes competency–course trees over a timeline. Pathology MKG research (BMC Med Ed 2025) advocates tiered graph exploration by user proficiency.

Past learnings: No docs/solutions/ entries in repo.

Topic Axes

  • Visualization UX & navigation — how users explore relational views beyond three parallel lists
  • Cross-curriculum relation graph — data model and API for edges between chunks, cases, keywords, frameworks
  • Overlap & redundancy detection — logic to find repeated teaching across cases
  • Gap vs overlap semantics — distinguishing external framework gaps from internal curricular duplication
  • Faculty vs student views — governance/review lens vs study-path lens on the same graph

1.Concept Bridge Graph

Replace the static three-column map with an interactive force-directed graph where curriculum sections are nodes and semantic similarity edges (pgvector cosine > threshold) connect content taught across different cases. Selecting a node highlights its framework alignments in a side panel; hovering an edge shows the shared concept snippet and both rationales. Implements the deferred AlignmentLines vision at course scale rather than selection-only beziers.

Visualization UX Relation graph Confidence 88% Complexity Medium
Basis
direct: chunks.embedding vector(1536) exists; map page has no edges (app/courses/[courseId]/map/page.tsx); MVP plan U8 lists AlignmentLines as deferred.
Rationale
Directly answers "show connections between curriculum elements" using data already computed at ingest — the highest-leverage gap in the current UI.
Downsides
Force layouts can overwhelm with 7×N chunks; needs clustering, threshold tuning, and performance guardrails. Similarity ≠ pedagogical redundancy without faculty confirmation.

2.Keyword Constellation Map

Surface the pipeline's AAMC keyword tags as a cross-cutting layer: keyword nodes sit between cases and frameworks, sized by how many case-sections share them. Users see "Ethics" or "Renal physiology" as explicit hubs linking Cases 2, 4, and 6 — a relational view orthogonal to USMLE/AAMC taxonomies. Clicking a keyword hub expands all aligned chunks and framework leaves.

Relation graph Overlap detection Confidence 85% Complexity Medium
Basis
direct: Pipeline inserts keyword_tags via retrieveKeywordCandidates (lib/pipeline.ts:154-169); zero UI or query consumption today.
Rationale
Keywords are institution-neutral cross-cutting concepts — exactly the "beyond USMLE/AAMC audit" layer the user requested, with data already in Postgres.
Downsides
Keyword tagging depends on framework embeddings; sparse tags weaken the graph. May duplicate Concept Bridge edges when keywords align with embedding clusters.

3.Framework Hub Lens with Redundancy Halo

Add a pivot mode to the existing map: select any USMLE subdomain or AAMC competency and the center panel becomes a hub showing all case-sections feeding it, with a redundancy halo when 3+ sections align above confidence threshold. Edge thickness encodes average confidence; color shifts from green (single coverage) to amber/red (over-taught). Extends FrameworkTree's binary green/gray without replacing the tri-directional layout faculty already know.

Visualization UX Gap vs overlap semantics Confidence 82% Complexity Low
Basis
direct: FrameworkTree only shows binary aligned state (components/map/FrameworkTree.tsx:21-31); gap_summary.chunkCount tracks per-framework counts but is not shown on map.
Rationale
Minimal UX delta on the signature feature; immediately surfaces overlap the gaps page cannot — "this competency is taught in five places."
Downsides
Still framework-centric; doesn't reveal cross-cutting themes outside taxonomy. Halo thresholds need calibration to avoid false alarms on intentional spiral curricula.

4.Overlap–Gap Dual Lens

Split the map into linked panels: left = framework gaps (existing gap_summary logic), right = redundancy clusters (framework leaves or keywords with high chunkCount). Selecting a gap on the left highlights related overlap nodes on the right — surfacing whether a "missing" topic is actually covered under adjacent competencies elsewhere. Annotates suggested actions: "add content" vs "consolidate Cases 3 & 5" vs "intentional spiral — no action."

Gap vs overlap semantics Faculty vs student views Confidence 78% Complexity Medium
Basis
direct: suggestedGapAction only recommends adding/strengthening content (lib/gap-analyzer.ts:17-24); mapEDU markets combined gap+redundancy reports as core value (external: mapedu.com).
Rationale
Reframes curriculum review as a two-sided audit — the user's explicit ask to go beyond framework alignment.
Downsides
Linking gaps to overlaps requires heuristic adjacency (shared parent framework or keyword); false links could mislead directors. Needs faculty override for "intentional repeat."

5.Chunk Similarity Redundancy Radar

Background job (at db:process or new db:analyze-overlap) computes top-k similar chunk pairs per course via pgvector, storing results in a chunk_similarity table. A "Redundancy Radar" tab ranks pairs by similarity × shared framework count, presenting side-by-side excerpts with diff highlighting. Faculty mark pairs as "consolidate," "differentiate," or "intentional spiral."

Overlap detection Relation graph Confidence 80% Complexity Medium
Basis
direct: searchChunks already runs pgvector cosine queries (lib/queries.ts:192-198); no cross-chunk pairwise analysis exists.
Rationale
Most precise overlap signal available without new AI calls; actionable list format suits faculty governance workflows.
Downsides
O(n²) pairwise cost for large corpora — needs batched top-k only. High similarity in different pedagogical contexts (intro vs advanced) produces false positives.

6.Competency Timeline Scrubber

ForestED-inspired horizontal timeline: x-axis = case sequence (1–7), y-axis = competency domains or cross-cutting keywords. Cells show first introduction vs reappearance of each concept, with saturation encoding depth (chunk count × confidence). Scrubbing reveals how the curriculum spirals — or accidentally repeats — across the semester. Complements the static dashboard heatmap with temporal relational context.

Visualization UX Faculty vs student views Confidence 75% Complexity Medium
Basis
external: ForestED (IADIS 2025) — tree/timeline visualization of competency development across curriculum sequence; direct: documents ordered by caseNumber in getMapData.
Rationale
Medical curricula are inherently sequential; temporal view makes "spiral vs redundant repeat" legible at a glance for course directors.
Downsides
Case order may not equal teaching order in F2F schedule. Dense y-axis with 100+ competencies needs rollup to domains.

7.What-if Case Trim Simulator

Non-destructive "Structure Explorer" mode (CloudPedagogy pattern): toggle cases off and watch gaps and redundancy metrics recalculate live. Shows which cases are load-bearing for coverage vs which are redundant contributors. Export a trim proposal for curriculum committee review. Operates on in-memory graph derived from existing alignments — no schema migration required for v1.

Faculty vs student views Gap vs overlap semantics Confidence 72% Complexity High
Basis
external: CloudPedagogy Structure Explorer — non-destructive what-if for curriculum refactoring; direct: gap recompute already runs at course level (recomputeCourseFrameworkGaps).
Rationale
Turns relational insight into governance action — "if we cut Case 5, which gaps open?" — differentiated from passive audit dashboards.
Downsides
Client-side recompute must mirror server gap logic exactly or trust an API. High complexity for demo scope; better as phase-2 after graph foundation (ideas 1–3).
