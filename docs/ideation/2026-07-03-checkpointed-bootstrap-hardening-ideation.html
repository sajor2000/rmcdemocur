Grounding Context

Resume mechanics (#1 incremental upsert, #2 JSONL cache, #4 skip-processed docs) are scoped in the parent ideation. This pass asks: how do we harden and audit so operators trust checkpoints and catch partial failures before burning Azure spend?

  • Scaffold on disk: lib/bootstrap-state.ts, lib/embedding-cache.ts, scripts/bootstrap.ts, partial process-documents.ts exports — not wired to package.json yet.
  • Still batch-embed: seed-frameworks.ts embeds all ~614 USMLE rows before any DB write — kill at 120/614 still loses work.
  • Weak complete check: isDocumentPipelineComplete only checks alignments exist, not chunk embeddings — partial pipeline can look "done".
  • No audit CLI: manifest, JSONL cache, and Neon counts can drift with no reconcile command.
  • Constraint: no new Drizzle tables or migrations — use existing stable_id + embedding columns as checkpoints.

Topic Axes

  • Framework durability — per-row embed + cache + skip-if-embedded
  • Completion truth — when is a document or framework phase actually done?
  • Operator audit — read-only reconcile manifest ↔ DB ↔ cache
  • Smoke gates — block full run until Case 1 passes domain spot-checks
  • Test & CI contracts — unit tests for resume predicates without live Azure

#1Incremental embed → upsert (no bulk delete on resume)

Framework durability Complexity: Medium

For each framework row: skip if DB already has stable_id with non-null embedding; else embed one text, upsert immediately. Only bulk-delete with --force.

for row in usmle: if db.has_embedding(stable_id): continue vec = cache.get(id) ?? await azure.embed(text) cache.append(id, vec) db.upsert_by_stable_id(row, vec) manifest.frameworks.usmle.embedded++
Basis
direct: scripts/seed-frameworks.ts:107-126 still batch-embeds then deletes all tables.
Rationale
Neon row is the checkpoint; re-run after kill continues from last persisted embed.
Downsides
Slower full refresh; stale rows need explicit --force.
Confidence
94%

#2Strict pipeline-complete predicate

Completion truth Complexity: Low

isDocumentPipelineComplete returns true only when: chunks exist, every chunk has a non-null embedding, and at least one alignment exists. Export helper getDocumentPipelineStatus for audit output (complete | partial-chunks | partial-embed | partial-align).

Basis
direct: process-documents.ts:91-107 only counts alignments — chunks-without-embeddings would skip incorrectly.
Rationale
Prevents false "skip complete" on half-finished pipeline runs.
Downsides
Re-align-only workflows need --force to reprocess.
Confidence
91%

#3Read-only db:audit-bootstrap reconcile CLI

Operator audit Complexity: Medium

Print side-by-side: manifest phase, DB embed counts per table, cache line count, per-document pipeline status for all 14 docs. Exit non-zero on drift or any partial doc when smoke/full expected.

USMLE: 412/614 embedded (DB) | 412 lines (cache) | manifest: 412/614 Docs: 3 complete | 1 partial-embed | 10 pending Smoke: NOT VERIFIED
Basis
reasoned: parent ideation #3 warns manifest can drift from DB if scripts run manually.
Rationale
Single operator command before/after long runs; CI can run in dry mode with mocked DB.
Downsides
Another script to maintain; must stay read-only.
Confidence
88%

#4Embedding cache model/dims fingerprint

Framework durability Complexity: Low

Store { stableId, model, dimensions, vector } in JSONL. On read, ignore entries when deploy model or AZURE_OPENAI_EMBEDDING_DIMENSIONS differs. Log cache invalidation count at seed start.

Basis
direct: lib/embedding-cache.ts stores only stableId+vector today.
Rationale
Prevents silent wrong-vector reuse after model upgrade.
Downsides
Full re-embed after model change (expected).
Confidence
90%

#5Smoke gate with USMLE GI domain spot-check

Smoke gates Complexity: Low

Extend verifySmoke() in bootstrap.ts: Case 1 faculty alignments must include usmle:gastrointestinal-system:% and must NOT use usmle:social-sciences:gastrointestinal-system (plan 005 parser fix validation).

Basis
direct: docs/plans/2026-07-03-005 — GI content was mislabeled under social-sciences.
Rationale
Catches bad framework seed before 13 more documents process.
Downsides
Case-1-specific; may need update if demo course changes.
Confidence
86%

#6Wire db:bootstrap:* + gitignore runtime artifacts

Operator audit Complexity: Low

Add npm scripts for smoke/full/status/audit. Gitignore data/bootstrap-state.json and data/frameworks/.embedding-cache.jsonl. Document bootstrap chain in docs/README.md.

Basis
direct: scripts/bootstrap.ts exists but package.json has no entry points.
Rationale
Discoverability and no accidental commit of local checkpoint state.
Downsides
None significant.
Confidence
95%

Recommended Ship Set

Implement #1 + #4 together in seed-frameworks.ts, #2 in process-documents.ts, #5 + #6 in bootstrap.ts + package.json, #3 as follow-up audit CLI. All without schema migration.

npm run db:bootstrap:smoke → push → frameworks(resume) → seed → case1 process → verifySmoke npm run db:audit-bootstrap → reconcile manifest / DB / cache (read-only) npm run db:bootstrap:full → only if smoke verified
