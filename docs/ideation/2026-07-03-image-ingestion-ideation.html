Image Ingestion for RUSHMAP

2026-07-03 · topic: image-ingestion · focus: self-study & faculty guide figures · mode: repo-grounded

Grounding

  • ~2,900 embedded images across 14 curriculum files (self-study guides dominate).
  • parseDocument uses mammoth.extractRawText / pdf-parse — images never enter the pipeline.
  • Surviving text: Answer Image:, Figure 1A, answer captions in prose, instructions (“move the X on Figure 2A”).
  • Chunker stoplists Answer Image / Provided image so labels stay inside real sections.
  • Plan 007 explicitly defers OCR/vision — separate effort from chunking/alignment work.
  • Guides state figures in slides/notes can appear on assessments — visual-only content is a coverage gap.

Principle

Store pixels for display; embed text for search and alignment. Same split as video/transcript — never put raw images into pgvector.

Ranked Ideas

1. Mine existing figure prose first

Confidence: 90% · Complexity: Low

Chunk and align the captions already in Word: Answer Figure 1A: …, rationale lines, figure titles. Audit for sections that reference a figure with no descriptive text nearby.

Basis (direct): Faculty guides (Cases 3–5) already carry answer text for many Answer Image / Figure N blocks in extractable prose.

Downside: Histology/radiology slides in self-study may have image-only content.

2. Figure registry + gap audit

Confidence: 85% · Complexity: Low–Medium

Parse labels (Answer Image:, Figure \d+[A-Z]?, Provided image) into a registry: case, section, label, has_caption_in_text. Extend audit-chunks.ts with “image-only reference” warnings.

Basis (direct): ~2,900 images invisible today; registry quantifies what prose cannot cover.

3. Extract DOCX media + link in UI

Confidence: 80% · Complexity: Medium

Unzip word/media/* (or mammoth image handler) → data/curriculum/media/{case}/{doc}-{n}.png. Attach figure_id / storage path on chunk or sibling media_assets row. UI shows image when chunk mentions the label.

Basis (reasoned): Enables faculty review and student “open figure” without vision cost; alignment still text-only until captions exist.

Downside: DOCX order ≠ figure labels — needs heuristic or manual manifest for stable IDs.

4. Selective vision/OCR on flagged figures only

Confidence: 70% · Complexity: High

Run Azure vision/OCR only on registry rows marked image-only (especially faculty Answer Image blocks). Write description → auxiliary text chunk → embed with breadcrumb Case › Section › Figure 1A (described).

Basis (direct): Plan 007 deferred full OCR; scoped pass avoids processing all 2,900 LWW textbook diagrams.

Downside: Cost, latency, medical diagram accuracy needs human spot-check.

5. Faculty caption bundle (ops path)

Confidence: 75% · Complexity: Medium (external)

Ask curriculum team for a sidecar manifest: figure label → official caption / alt text (CSV or JSON). Prefer human captions over vision for assessment-critical images.

Basis (reasoned): Faculty already author answer rationales; many images are pedagogically labeled in guides.

Rejected

  • Multimodal image embeddings — stack is text RAG; no ROI for curriculum map.
  • OCR all 2,900 images upfront — cost/noise; many are duplicate textbook art with prose parity elsewhere.
  • Replace text pipeline with HTML layout parse — plan chose lexical chunking first; HTML is a separate path for PDF/DOCX parity.

Recommended sequence

  1. #1 prose mining + #2 registry/audit (ship together)
  2. #3 extract & link for UI on faculty Answer Image set first
  3. #5 caption bundle from curriculum ops, then #4 vision only on remaining gaps

Pair with video work

Shared media_assets shape: type (figure | video), label, case_id, section, url_or_path, text_for_embed (caption/transcript/description). One pipeline stage can ingest text sidecars; one UI component can render linked media.
